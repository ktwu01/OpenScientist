## Fact
Scientific conversation mining works better when extracted know-how is partitioned into three distinct memory types: procedural impasse rules, semantic missing facts, and episodic turning points. Engineering episodes should be rejected even if they occurred inside a research project. Within this taxonomy, procedural impasse rules deserve the highest priority because they are both the hardest to recover and often the most decision-relevant.

## Evidence
The session’s extraction framework enforced exactly these three categories and repeatedly distinguished scientific reasoning from engineering/debugging content during classification, cleaning, and scoring. The associated extraction guidance also treated procedural impasse rules as the highest-value memory class relative to semantic facts and episodic cases.

## LLM Default Belief
An LLM often defaults to treating any useful lesson from a research repository as “research knowledge,” collapsing methodology, factual discoveries, and implementation incidents into one undifferentiated set.

## Expiry Signal
This should be revised if later evaluations show another memory taxonomy produces better precision or retrieval for scientific reasoning tasks than the three-way split.
