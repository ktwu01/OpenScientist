## When
Time spent optimizing systems, tools, or workflows exceeds time producing results.

## Decision
Preferred: Shift to **output-first constraint** (papers, results, derivations).

Rejected:
- Further system refinement → no direct research gain.
- Tool comparison loops → infinite optimization.

Reasoning:
Only outputs generate feedback from reality.

## Local Verifiers
- No tangible outputs after extended setup period.
- Increasing system complexity without results.

## Failure Handling
If output quality is low:
- Iterate on output, not system.
- Use feedback cycles to refine.

## Anti-exemplars
- Early-stage infrastructure setup required for data collection.
